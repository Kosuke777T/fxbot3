## 1. トレーリングストップの問題 [優先度: 高]

### 1.1 retcode=10025 (No changes) の原因

**症状:** MT5がSL更新リクエストを `TRADE_RETCODE_NO_CHANGES (10025)` で拒否する

**根本原因:** SLをfloat生値のまま送信しており、MT5が内部でティックサイズに正規化した結果、
送信値と現在SLが同値になってしまっている。

例:
```
sl=156.05780473185436  # 小数17桁のfloat
sl=157.33590235949336  # MT5正規化後 → 前回SLと同値
```

**対策:**
```python
import MetaTrader5 as mt5

def normalize_price(symbol: str, price: float) -> float:
    """MT5のティックサイズに合わせて価格を正規化する"""
    info = mt5.symbol_info(symbol)
    if info is None:
        return round(price, 5)
    tick_size = info.trade_tick_size
    digits = info.digits
    normalized = round(round(price / tick_size) * tick_size, digits)
    return normalized
```

SL計算後、`modify_position()` に渡す前に `normalize_price()` を適用する。

### 1.2 トレーリングの活性化タイミング問題

**症状:** trailing exit が小損失で終了するケースが多数

例:
- EURJPY sell 182.82 → trailing exit 182.826 (PnL: -42) ← original SL 182.923より改善しているが損失

**分析:** トレーリングは機能しているが、利益が乗る前に相場が反転している。
現状の活性化条件（利益pips到達後にトレーリング開始）が適切かどうか再検討が必要。

**対策候補:**
- トレーリング活性化の利益閾値を引き上げる (例: 10pips → 20pips)
- ATRベースのトレーリング幅に変更する
- バックテストでトレーリングパラメータを最適化する

## 2. モデル性能の問題 [優先度: 高]

### 2.1 勝率低下 (35% < 閾値40%)

**症状:** `モデル勝率35%が閾値40%を下回っています` の警告が継続発生

**要因:**
- 学習データ期間: 58日 (WFO要求の180+30日に対して大幅不足)
- WFOウィンドウが自動縮小されており、モデル品質が低下
- 相場環境の変化にモデルが追随できていない可能性

**対策:**
- データ収集期間を延長する (少なくとも180日分)
- 勝率閾値を動的に調整する、またはデータ不足時は取引停止とする
- 再学習頻度を見直す

### 2.2 confidence常に1.0 (フィルター無効化疑い)

**症状:** 全37件のトレードで `confidence = 1.0`

**分析:** 信頼度スコアが常に最大値を返している。以下の原因が考えられる:
1. `MarketFilterConfig.confidence_threshold` が0以下に設定されている
2. `generate_signal()` の信頼度計算ロジックにバグがある
3. モデルの出力確率が常に高い値になるよう過学習している

**調査手順:**
```python
# config/default_settings.yaml を確認
# market_filter.confidence_threshold の値を確認

# signal.py の信頼度計算箇所を確認
# FilterStatus(filter_name="confidence", ...) の current_value を確認
```

**対策:**
- `default_settings.yaml` の `confidence_threshold` 設定値を確認・修正
- ログに信頼度スコアの実値を出力して追跡する

## 3. インフラ/接続の問題 [優先度: 中]

### 3.1 Slack SSL証明書エラー

**症状:** 2026-03-01以降、Slack通知が継続的に失敗

```
SSLError: [SSL: CERTIFICATE_VERIFY_FAILED] certificate verify failed
```

**対策:**
```bash
# 証明書の更新
pip install --upgrade certifi

# または環境変数で証明書パスを指定
export SSL_CERT_FILE=$(python -c "import certifi; print(certifi.where())")
```

Windows環境の場合、システム証明書ストアを使用するよう設定変更も検討。

### 3.2 MT5認証エラー

**症状:** 2026-03-01に `MT5 initialize() failed` が発生

**対策:**
- MT5ターミナルの自動起動設定を確認
- 認証情報 (login/password/server) が `default_settings.yaml` に正しく設定されているか確認
- MT5ターミナルのセッションタイムアウト設定を延長

## 4. WFOデータ不足 [優先度: 中]

**症状:** `データ期間58日がWFOの要求(180+30=210日)に対して不足`

**影響:** WFOウィンドウが自動縮小され、アウトオブサンプル検証精度が低下

**対策:**
1. **短期対応:** WFOパラメータをデータ量に合わせて調整
   ```yaml
   wfo:
     train_days: 40   # 180 → 40
     test_days: 10    # 30 → 10
   ```
2. **中長期対応:** 1〜2ヶ月データ蓄積を継続し、パラメータを元に戻す
