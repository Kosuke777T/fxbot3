# fxbot3 機能評価

作成日: 2026-03-07

## 前提

このリポジトリ内には `ミチビキ` という名称の仕様書やコードは見当たりませんでした。
そのため本評価では、以下を「ミチビキとして作ろうとしていたソフトの構想・意図」に相当する資料として扱います。

- `docs/要件.txt`
  - MT5 API を使った Python 製 FX 自動売買
  - AI 予測でのエントリー
  - SHAP の活用
  - 設定タブでのデモ/リアル口座切替
- `docs/概要.txt`
  - 初期段階の 7 フェーズ実装構想
  - 当初は 5 タブ中心の GUI
- `改善点.md` `改善点2.md` `改善点3.md`
  - 実運用後に見えてきた追加したい機能や改善候補

したがって本書の比較は、厳密な「別プロジェクトの完成版比較」ではなく、`fxbot3` と「ミチビキ構想/改善案」の比較として読むのが正確です。

## 結論

`fxbot3` は、初期構想レベルのミチビキよりかなり先まで進んでいます。
特に、単なる AI 売買ツールではなく、以下まで含んだ「運用・分析付きの売買システム」になっています。

- MT5 接続とライブ取引
- LightGBM 学習
- SHAP 分析
- WFO 付きバックテスト
- 取引ログ保存
- 市場フィルター
- 複数通貨ペア運用
- 全体監視
- 戦略分析
- 戦略アドバイザー
- 週末自動再学習

一方で、運用成績の改善に直結する機能の一部は、まだ「実装済みでも使い切れていない」または「構想止まり」の部分が残っています。
つまり `fxbot3` は機能量ではミチビキ構想を上回るが、収益最適化と運用安定性では未完成要素が残る、という評価です。

## 1. fxbot3 の現行機能評価

### 1-1. コア売買機能

実装済み機能は十分に広いです。

- MT5 接続、シンボル自動検出、注文送信、変更、決済
- M5 ベースのライブ取引ワーカー
- 複数時間足特徴量
- LightGBM 学習
- 回帰/分類の両モード対応
- SHAP による特徴量選択
- バックテスト
- WFO
- ATR ベースの SL/TP/トレーリング
- ロット計算、ポジション数制御
- モデル劣化監視
- 週末自動再学習
- Slack 通知

評価:

- 売買エンジンとしての機能密度は高い
- 単発の売買ロジックではなく、学習から運用まで一通り閉じている
- 初期要件にあった MT5 + AI + SHAP + 口座切替はすべて満たしている

### 1-2. GUI と運用支援

初期構想では 5 タブ中心でしたが、現行 `fxbot3` は 12 タブ構成まで拡張されています。

- ダッシュボード
- 通貨ペア
- 一括学習
- モデル
- バックテスト
- SHAP
- 設定
- 市場フィルター
- 取引ログ
- 通貨別成績
- 全体監視
- 戦略分析

評価:

- この拡張が `fxbot3` の最大の強み
- 取引するだけでなく、「何が起きているか」「なぜ負けているか」を画面で追える
- 初期のミチビキ構想より、運用者向けの可観測性がかなり高い

### 1-3. 分析機能

特に強いのは分析系です。

- 取引ログの SQLite 保存
- 通貨ペア別成績比較
- 全体監視タブでの稼働状態確認
- 戦略分析タブでの HOLD 理由、フィルター通過率、方向別成績、時間帯別成績、予測値帯別成績、モデル別成績の可視化
- ルールベースの戦略アドバイザー

評価:

- ここはミチビキ構想より明確に先行している
- 「勝率を上げるためにログを残して利用したい」という `docs/推敲.txt` の意図を、かなり具体的に実現できている
- `fxbot3` は分析基盤付きの売買ソフトとして評価できる

## 2. ミチビキ構想との差分

### 2-1. fxbot3 が上回っている点

`ミチビキ` の初期構想と比較した場合、`fxbot3` が上回っている点は次の通りです。

| 項目 | ミチビキ構想 | fxbot3 |
| --- | --- | --- |
| AI 売買 | あり | あり |
| SHAP | あり | あり |
| 口座切替 | あり | あり |
| バックテスト | 構想上は想定されるが資料上は薄い | 実装済み |
| WFO | 明示なし | 実装済み |
| 取引ログDB | 発想あり | 実装済み |
| 複数通貨ペア | 明示なし | 実装済み |
| 市場フィルター | 明示なし | 実装済み |
| モデル劣化監視 | 明示なし | 実装済み |
| 自動再学習 | 明示なし | 実装済み |
| 全体監視画面 | 明示なし | 実装済み |
| 戦略分析画面 | 明示なし | 実装済み |
| 戦略アドバイザー | 明示なし | 実装済み |

要するに、`fxbot3` はミチビキ構想の「基礎機能」を超えて、運用改善ループまで持ち込んでいます。

### 2-2. 差はあるが、強みである一方で複雑化もしている点

`fxbot3` は高機能化した反面、次の複雑さも抱えています。

- 設定項目が多く、最適値の維持が難しい
- 実運用データ不足時に WFO や再学習の信頼性が落ちる
- トレーリング、閾値、通貨ペア差などの調整余地が大きい
- 「機能はあるが、成績改善にまだ十分つながっていない」箇所がある

つまり、ミチビキがもし「シンプルで扱いやすい AI 売買ソフト」を目指していたなら、`fxbot3` はそれより高性能だが、そのぶん調整前提のシステムになっています。

## 3. ミチビキで入れようとしていた機能のうち、fxbot3 に入ったもの

リポジトリ内の資料から見て、ミチビキ側で志向していた機能で、`fxbot3` に反映されたものは次の通りです。

- MT5 API を用いた Python 製自動売買
- AI 予測によるエントリー
- SHAP を使った特徴量評価
- デモ/リアル口座切替
- 売買結果ログの保存
- ログを使った勝率改善のための分析

特に「売買結果のログを残して勝率を上げるために利用したい」という意図は、以下の形で具体化されています。

- `取引ログ` タブ
- `通貨別成績` タブ
- `戦略分析` タブ
- `全体監視` タブ
- モデル劣化監視
- 戦略アドバイザー

この点では、`fxbot3` はミチビキの問題意識をかなり真面目に継承していると評価できます。

## 4. ミチビキで入れたかったが、fxbot3 で未完成または未実装と見なせる機能

改善資料から見ると、まだ弱い部分は次の通りです。

### 4-1. 成績改善に直結する未完成要素

- 通貨ペア別の個別閾値調整
  - 例: EURJPY だけ閾値やスプレッド条件を厳しくする
- クラス不均衡対策
  - `class_weight` などの最適化余地
- WFO 不合格時の明確な停止/抑制ルール
- データ量不足時の再学習評価の扱い
- 運用成績に応じた自動パラメータ最適化

評価:

- これらは「分析して終わり」でなく「分析結果を自動で設定へ反映する」段階
- ミチビキが目指していたであろう「勝率向上支援」は、まだ半自動止まり

### 4-2. 可視化でまだ拡張余地があるもの

`docs/説明書3.md` では、今後の拡張候補として以下が挙がっています。

- 日別損益グラフ
- 曜日別成績
- フィルター別損益比較
- 戦略イベントの長期推移グラフ
- モデル切替前後の比較表示

評価:

- 現在でも分析機能は強いが、長期推移や比較可視化はまだ不足
- ミチビキが「判断支援ツール」寄りを目指すなら、この領域は今後の伸びしろ

### 4-3. 運用安定性で残る課題

改善資料で繰り返し出ている課題:

- WFO に対するデータ期間不足
- Slack SSL エラー
- MT5 認証/接続エラー
- AutoTrading 停止やパラメータ不整合

評価:

- 売買ロジック以前に、運用基盤の安定化が未完全部分として残っている
- ミチビキ比較というより、実戦投入品質の課題

## 5. 総合評価

総合すると、`fxbot3` は機能面ではかなり強いです。

### 強い点

- 初期構想を大きく上回る機能量
- 売買、学習、バックテスト、再学習、監視、分析が一体化している
- ログ活用による改善サイクルが既に作られている
- GUI が運用者向けにかなり実用的

### 弱い点

- 高機能ゆえに設定と検証が難しい
- 成績改善の核心部分はまだ調整中
- データ不足や接続エラーなど、運用品質の弱点が残る
- 一部の改善案は資料上で提案済みだが、完全にはシステム化されていない

### 最終判断

`fxbot3` は「ミチビキとして考えていた AI 売買ソフト」の上位発展版と評価できます。
ただし、完成度の評価を分ける必要があります。

- 機能完成度: 高い
- 分析/可視化完成度: 高い
- 収益改善の自動化完成度: 中程度
- 実運用安定性: 中程度

したがって、機能面の比較では `fxbot3` が優位です。
一方で、「勝率を上げ続けるための自律的な改善システム」としては、まだミチビキ的な理想像に完全到達していない、というのが妥当な評価です。
